磁盘配额

磁盘配额概念

磁盘配额是计算机中指定磁盘的储存限制，就是管理员可以为用户所能使用的磁盘空间进行配额限制，每一用户只能使用最大配额范围内的磁盘空间。

磁盘配额条件

  • 内核必须支持磁盘配额
  • 系统中必须安装了 quota 工具，我们的Linux默认是安装了 quota 工具的，查看命令如下：

要支持磁盘配额的分区必须开启磁盘配额功能，这个功能需要手工开启，不再是默认开启。

概念

1）用户配额和组配额
2）磁盘容量限制和文件个数限制
3）软限制和硬限制
4）宽限时间

如果用户的空间占用数处于软限制和硬限制之间，系统会在用户登陆时警告用户磁盘将满，这个时间就是宽限时间，默认是7天。如果达到了宽限时间，用户的磁盘占用量还超过软限制，那么软限制就会升级成为硬限制。

磁盘配额规划

我们开始磁盘配额实验，首先来规划一下实验：

  • 磁盘配额是限制普通用户在分区上使用磁盘空间和文件个数的，所以需要指定一个分区。那么我们手工新建一个5GB的/dev/sdb1分区，把它挂载到/disk目录当中。
  • 还有需要建立被限制的用户和用户组。那么我们假设需要限制 user1、user2和user3用户，这三个用户属于 test 用户组。
  • 其中 test 组磁盘容量硬限制为500MB，软限制为450MB，文件个数不做限制。user1用户位来便于测试，磁盘容量硬限制为50MB，软限制为40MB，文件个数限制为10个，软限制为8个。user2和user3用户磁盘容量硬限制为250MB，软限制为200MB，文件个数不做限制。
  • 这时会发现三个用户加起来的磁盘容量限制为550MB，超过了 test 组的磁盘容量限制500MB。这样的话，某个用户可能达不到自己的用户限制，而达到组限制时就不能再写入数据了。也就是说，如果用户限制和组限制同时存在，那么哪个限制小，哪个限制优先生效。
  • 系统宽限时间改为8天。
