正则表达式

概述

正则表达式用来匹配符合条件的字符串，是包含匹配。在文件中搜索字符串的命令，如grep、awk、aed(文本三剑客) 等命令可以支持正则表达式。
通配符是用来匹配符合条件的文件名，是完全匹配。用ls、find、cp 这些命令进行匹配。

基础正则表达式

元字符 作 用
* 前一个字符匹配0次或任意多次
. 匹配除了换行符外任意一个字符
^ 匹配行首。例如：^hello会匹配以hello开头的行
$ 匹配行尾。例如：hello$会匹配以hello结尾的行
[ ] 匹配中括号中指定的任意一个字符，只匹配一个字符。例如：[aeiou]匹配任意一个原因字母。
[^] 匹配除中括号内字符以外的任意一个字符。例如：[^0-9]匹配任意一位非数字字符。
\ 转义符。用于取消讲特殊符号的含有取消。
\{n\} 表示其前面的字符恰好出现n次。例如：[0-9]\{4\} 匹配4为数字，[1][3-8][0-9]\{9\} 匹配手机号
\{n,\} 表示其前面的字符出现次数不小于n次。例如:[0-9]\{2,\} 表示两位及以上的数字。
\{n,m\} 表示前面的字符至少出现n次，最多出现m次。例如：[a-z]\{6,8\} 匹配6到8的小写字母。

扩展正则表达式

正则表达式中还可以支持一些元字符，比如"+" “?” “|” “( )”。Linux是支持这些元字符的，只是grep命令默认不支持，要想要支持这些字符，必须使用egrep命令或grep -E选项，所以我们又把这些元字符称作扩展元字符。
如果查询grep的帮助，对egrep的说明就是grep -E 选项一样的命令，所以我们可以把两个命令当中别名来区别对待。Shell中支持的扩展元字符：

扩展元字符 作 用
+ 前一个字符匹配1次或任意多次。如果 "go+gle" 会匹配 "gogle" 、"google" 或"gooogle",当然如果"o"有多个，也能匹配。
? 前一个字符匹配0次或1次。如"colou?r" 可以匹配"colour"或"color"。
I 匹配两个或多个分支选择。如"was|his"会匹配既包含"was"的行，也可以匹配"his"的行
( ) 匹配其整体为一个字符，即模式单元。可以理解为由多个单个字符组成的大字符。如"(dog)+"会匹配成"dog"、"dogdog"、"dogdogdog"等，因为被（）包含的字符会当成一个整体。但"helllo(world|earth)"会匹配"hello world"及"hello earth"。

awk流程控制

举例：在awk中定义变量与调用变量的值来统计PHP成绩的总分。

我们解释下这个命令。“NR2{php1=$3}”（条件是NR2，动作是php1=$3）这句话是指如果输入数据是第二行（第一行是标题行），就把第二行的第三字段的值赋予变量“php1”。“NR3{php2=$3}”这句话是指如果输入数据是第三行，就把第三行的第三字段的值赋予变量“php2”。“NR4{php3=$3;totle=php1+php2+php3;print "totle php is " totle}”（“NR==4”是条件，后面{}中的都是动作）这句话是指如果输入数据是第四行，就把第四行的第三字段的值赋予变量“php3”；然后定义变量totle的值是“php1+php2+php3”；然后输出“totle php is”关键字，后面加变量totle的值。
在awk编程中，因为命令语句非常长，在输入格式时需要注意以下内容：

  • 多个条件{动作} 可以用空格分隔，也可以用回车分隔。
  • 在一个动作中，如果需要执行多个命令，需要用" : "分隔，或者用回车分隔。
  • 在awk中，变量的赋值与调用都不需要加 $ 符。
  • 条件中判断两个值是否相同，请使用"==" ,以便于区分。
